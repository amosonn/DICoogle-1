2.2 Functional definitions of Application Entity

Storage Server

RSI Storage Server waits for another application to connect at a configured TCP/IP port. When another application makes a DICOM association request, RSS acts a SCP for the storage service class: stores the file on disk and indexes it's fields to allow simplified search in received files.

3. AE Specifications

3.1 Storage Server AE Specification

Association Establishment Policies

RSS accepts association in response to C-STORE requests from other AE. It will only accept C-STORE requests for SOP classes that has been configured to accept by the user. This means that even if the requested SOP class is supported by RSS if the user chose to not to allow RSS to accept it, then RSS will reject all C-STORE requests for that SOP class. It can also be configured to only accept requests from a group of chosen AE, that are identified by its AET, and to reject any request that does not target his own configured AET. When it accepts a C-STORE request it will act as a storage service class provider. It also accepts association in response to C-ECHO requests and on that case act as verification services class provider.

RSS does not limit the number of simultaneous associations that will be accepted, but the TCP/IP protocol stack does have a finite number of available ports that act as limit, although that limit should never be reached.

RSS by itself never initiates an association.
